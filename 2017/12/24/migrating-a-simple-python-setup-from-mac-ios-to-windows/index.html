Migrating A Simple Python setup from Mac iOS to Windows

2017-12-24 · 257 words · 2 minute read
python · windows · mac · iOS

Recently I purchased an Acer gaming laptop loaded with Windows 10 to play games on. I was really attracted to the quality of the hardware vs the price. The hard part would be moving off of the Unix type environment I’ve been used to. Adjusting to Windows took a little tweaking on my part. My goal was to get my workflow as similar as possible with the least amount of effort. I didn’t want to have to deal with the Windows/Unix shell.

Started getting my Python setup going

I started by downloading the python-2.7.14.msi file from https://www.python.org/downloads/ and completing the wizard. Once Python was installed the next step was to figure out how to get going with Pip

Using the default setup on windows, to perform Python incantations

python -m

Such as to invoke pip

python -m pip install virtualenv

Or when spinning up a virtualenv

python -m virtualenv venv

But you don’t have to use the -m flag to run a script

python my_script.py

and since I learned that virutalenv knew this was going to be a problem that added a .bat file inside of that venv we just created so to activate it just go and run venv\Scripts\activate from inside of the current directory where the venv was created.

You can use git bash to do a bunch of this stuff to so it is much more Unix style When you open git bash run as administrator and make a new .bashrc file in the ~ directory

include things there like export PATH=$PATH:/c/Python27/ export PATH=$PATH:/c/Python27/Scripts
